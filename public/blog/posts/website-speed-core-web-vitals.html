سئو تکنیکال

سرعت سایت و Core Web Vitals؛ چرا مشتری را از دست می‌دهید

تحقیقات گوگل نشان می‌دهد به ازای هر ثانیه تأخیر در بارگذاری موبایل، نرخ ترک صفحه ۳۲٪ بالا می‌رود. اگر سایت‌تان کند است، نه‌تنها رتبه گوگل پایین‌تری دارید، بلکه مشتری‌هایی که به سختی جذب کرده‌اید را هم از دست می‌دهید. در این مقاله یاد می‌گیرید Core Web Vitals چیست، چطور سایت‌تان را بسنجید و کجا شروع کنید.

نویسنده: تیم آرایه ۷ تیر ۱۴۰۵ ۱۱ دقیقه مطالعه

از سال ۲۰۲۱ گوگل معیارهای تجربه کاربری را به‌عنوان فاکتور رتبه‌بندی لحاظ کرده است. این معیارها را «Core Web Vitals» می‌نامند: سه عدد ساده که سرعت، پایداری بصری و واکنش‌گری سایت‌تان را اندازه می‌گیرند. سایتی که این اعداد را خوب دارد، هم گوگل دوستش دارد، هم کاربر.

۱. Core Web Vitals را بشناسید

سه معیار اصلی اینها هستند:

LCP — بزرگ‌ترین محتوای قابل مشاهده

Largest Contentful Paint زمانی است که بزرگ‌ترین عنصر قابل مشاهده صفحه (معمولاً تصویر اصلی یا تیتر بزرگ) بارگذاری می‌شود. هدف: زیر ۲.۵ ثانیه. بالای ۴ ثانیه ضعیف است.

INP — واکنش به تعامل

Interaction to Next Paint (جایگزین FID از ۲۰۲۴) سرعت پاسخ سایت به کلیک، لمس یا تایپ کاربر را می‌سنجد. هدف: زیر ۲۰۰ میلی‌ثانیه.

CLS — جابه‌جایی چیدمان

Cumulative Layout Shift میزان جابه‌جایی ناخواستهٔ عناصر صفحه حین بارگذاری را اندازه می‌گیرد. مثلاً وقتی دکمه‌ای را می‌خواهید بزنید و ناگهان صفحه تکان می‌خورد. هدف: زیر ۰.۱.

ابزار رایگان: PageSpeed Insights

آدرس سایت‌تان را در pagespeed.web.dev وارد کنید تا امتیاز Core Web Vitals و لیستی از مشکلات اولویت‌بندی‌شده ببینید. حتماً هر دو نسخه موبایل و دسکتاپ را بررسی کنید؛ موبایل معمولاً مشکل‌دارتر است.

۲. دلایل اصلی کندی سایت در ایران

علاوه بر مشکلات جهانی، چند عامل خاص کسب‌وکارهای ایرانی وجود دارد:

  • تصاویر فشرده‌نشده: اکثر سایت‌های ایرانی تصاویر ۳–۵ مگابایتی آپلود می‌کنند؛ باید زیر ۲۰۰ کیلوبایت باشند.
  • فونت‌های فارسی سنگین: فونت Vazirmatn یا Shabnam را به‌صورت WOFF2 و فقط وزن‌های موردنیاز بارگذاری کنید.
  • سرور با پینگ بالا: هاست ایرانی با CDN مناسب یا هاست اروپایی نزدیک انتخاب کنید.
  • پلاگین‌های اضافه وردپرس: هر پلاگین چندین فایل JS/CSS بار می‌کند؛ فقط ضروری‌ها را نگه دارید.
  • رندر جاوااسکریپت مسدودکننده: اسکریپت‌هایی که صفحه را block می‌کنند باید defer یا async باشند.

۳. اقدامات عملی برای سرعت بیشتر

اینها را به ترتیب اولویت انجام دهید:

  1. تصاویر: همه عکس‌ها را به فرمت WebP تبدیل و فشرده کنید. از loading="lazy" برای عکس‌های پایین صفحه استفاده کنید.
  2. فونت: فونت فارسی را preload کنید و از font-display: swap استفاده کنید تا متن قبل از لود فونت نمایش داده شود.
  3. CSS و JS: فایل‌ها را minify و ترکیب کنید؛ کد استفاده‌نشده را حذف کنید.
  4. Caching: برای فایل‌های ثابت (عکس، CSS، JS) هدرهای cache طولانی تنظیم کنید.
  5. CDN: از شبکه توزیع محتوا استفاده کنید تا فایل‌ها از نزدیک‌ترین سرور به کاربر بارگذاری شوند.
  6. LCP image را preload کنید: تصویر اصلی صفحه (hero) را با تگ <link rel="preload"> اولویت‌دهی کنید.

۴. CLS را چطور درست کنیم؟

جابه‌جایی چیدمان اغلب از این موارد می‌آید:

  • تصاویر بدون width و height: حتماً اندازه را در HTML مشخص کنید تا مرورگر جا نگه دارد.
  • آگهی‌ها و بنرهای پویا: یک ظرف با ارتفاع ثابت برایشان رزرو کنید.
  • فونت‌هایی که جایگزین می‌شوند: font-display: optional یا swap با اندازه مناسب fallback font.
  • محتوای بارگذاری‌شده دیرهنگام: اگر محتوا از API می‌آید، skeleton placeholder نمایش دهید.
سرعت سایت تنها یک فاکتور سئو نیست — یک عامل احترام به کاربر است. سایت کند یعنی وقت مشتری برایتان مهم نیست.

۵. ابزارهای اندازه‌گیری

برای پایش مداوم این ابزارها را در نظر بگیرید:

  • PageSpeed Insights: سریع‌ترین راه برای گرفتن امتیاز Core Web Vitals.
  • Google Search Console: بخش Core Web Vitals نشان می‌دهد کدام صفحه‌ها مشکل دارند.
  • WebPageTest: آنالیز عمیق‌تر با waterfall chart.
  • Chrome DevTools: تب Performance برای بررسی دقیق در مرورگر.

جمع‌بندی

سایت سریع = رتبه بهتر + مشتری بیشتر + نرخ ترک کمتر. این سه، با هم، مستقیماً به درآمد تبدیل می‌شوند. در آرایه، سرعت بالا و نمره Core Web Vitals خوب بخشی از استاندارد پیش‌فرض هر پروژه است؛ نه یک گزینهٔ اضافه. برای ترکیب کامل، چک‌لیست سئوی سایت کسب‌وکار ما را هم بخوانید.

سایت‌تان امتیاز PageSpeed پایینی دارد؟

تیم آرایه سرعت و تجربه کاربری سایت شما را بررسی و بهینه می‌کند تا Core Web Vitals سبز شود.
