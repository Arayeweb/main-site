## بخش‌های بررسی در Search Console

### ۱) ایندکس و پوشش (Indexing / Pages)

- آیا صفحات بالا ایندکس شده‌اند؟
- آیا صفحات با `noindex` اشتباهی ایندکس نشده‌اند؟
- وضعیت sitemap (در صورت ثبت): آیا صفحه‌ها را کشف کرده؟

### ۲) Performance (Impressions / Clicks / CTR)

- کدام صفحات impressions دارند؟
- CTR (برای صفحات با کلیک) چگونه است؟
- average position برای صفحات مهم چیست؟
- صفحات با zero impressions کدام‌اند؟

### ۳) Crawl / Errors

- crawl errors داریم؟
- Redirect یا 404 در مسیرهای مهم دیده می‌شود؟

### ۴) لید از کانال آلی/سئو (در سیستم فرم/لید)

- از کدام صفحات/توابع، فرم یا لید ثبت شده؟
- آیا لیدهای ثبت‌شده به `page/source` درست نگاشت شده‌اند (خصوصاً `free-seo-audit`)؟

## قوانین تصمیم‌گیری (Decision Rules)

1. اگر یک صفحه impressions دارد ولی CTR پایین است:
   - عنوان و meta description را دقیق‌تر و مطابق نیت جستجو کنید.
   - از CTA/اسنیپت روشن‌تر استفاده کنید (بدون کلی‌گویی).

2. اگر یک صفحه بعد از ۲ تا ۴ هفته impressions ندارد:
   - لینک داخلی را تقویت کنید (از صفحات مهم به همان صفحه).
   - عمق محتوای همان صفحه را با سوالات واقعی کاربران بیشتر کنید.

3. اگر clicks دارید ولی لید/فرم ندارید:
   - CTA و محل فرم را به مسیر «تماس/رزرو» نزدیک‌تر کنید.
   - سرعت و UX موبایل را دوباره بررسی کنید.

4. اگر یک موضوع از ۴ مقاله وبلاگ traction گرفت:
   - فقط برای همان موضوع ۲ مقاله پشتیبان دیگر بسازید (نه انبوه و نه اسپم).

## قانون ضد اسپم و ضد گسترش زودهنگام

- تا وقتی حداقل سیگنال مفید از ۴ صفحه indexable نگرفتیم، ۱۲ صفحه‌ی noindex برنامه‌ریتی را وارد ایندکس نکنید.
